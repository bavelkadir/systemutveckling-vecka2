Fråga 3:

GitHub som det låter är en ”Hub” man kan se det lite som en gemenskap plattformen erbjuder utvecklare att lagra sin kod, där dom samarbetar i realtid och lite som git man kan hantera version historiker. Den har en koppling till ”Git” alltså ett versionshateringssystem såsom git så kan du spåra ändringar, arbeta parallellt med ditt team och jobba med koden eller projektet på ett strukturerat sätt.

Med olika brancher så är det en separat version, man kan se det lite som grenar i träd där själva stommen är (main versionen) så varför skapar man brancher, jo vi skapar brancher om vi ska tillexempel fixa någon ny funktion, fixa någon bugg eller testa något nytt. På så sätt så kan vi leka lite mer fritt än att skada själva main branchen.

En pull request är en funktion som tillåter att du får integrera ändringar från en branch till en annan då brukar det oftast vara en vanlig branch till main. Detta kan vara då tillexempel som, vi har idag gjort några ändringar kan ni titta på detta och godkänna ifall det ser bra ut. I detta så granskas koden, någon form av diskussion sker och tester innan man merge koden men det kommer jag ta längre ner.

Merge är som det låter att slå ihop, detta innebär ifall en pull request skulle bli godkänt så merge man ändringar in i själv main branchen det är lite som att sätta ihop två saker och det blir till en enda produkt.
