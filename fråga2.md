Fråga 2:

Git är ett verktyg som sparar alla ditt projekt som du kan alltid återblicka och gå tillbaka till koden när önskar att göra det. I Git så kan du arbeta utan bli störd av någon annan i ditt team som tillhör samma version av projektet. Du kan alltid spåra historik om du har gjort något fel för att sedan återskapa tidigare version, inte nog med så kan man även slå ihop ändringar som sker från olika i ditt team på ett smart om smidigt sätt. Man kan se Git som en mapp som organiserar alla projekt ”repository” du kan klona dessa repository vilket gör att utvecklarna i ditt team kommer kunna få en kopia av projektet och historian sparas på er egen dator. Därför är det viktigt med git då projektet blir mycket enklare att lösa.

En live scenario: 

En live scenario kan innebära om det så att vi har skapat en app säg tillexempel en liknade app som Facebook och vi utvecklar själva sökmotorn eller sökfältet via appen och så gör vi en commit på det. Men dagen efter så märker du att sökfältet inte fungerar som det ska någon i ditt team har förmodligen skriver över din kod eller råkat göra något under en buggfix, med git så kan du lösa det problemet enkelt genom att göra en ”revert” så utan git så hade du behövt skriva om hela koden.
